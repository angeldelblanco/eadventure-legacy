Adaptation Profile

The way of generating the adaptation profiles is very similar to the generation of assessment profiles. These are used to adapt the behavior of game based on a set of "external properties" coming from the server of learning (state of the LMS).

When these properties are fulfilled at the beginning of the chapter a set of flags and initial variables are activated or deactivated automatically and first scene changes (if desired). So, it is possible to change the number of available objects, adapt the difficulty according to the student profile, etc.

Also these external properties can be empty in order to make changes in game automatically at startup. This way it is possible to provide differents version of the game simply by changing the adaptation profile associated with it.

In the same way that in assessment profiles, the type of adapatation profile must be set as SCORM 2004 or SCORM v1.2 if we are going to make an exportation of this type. If you select any type, a list will appear in the attribute "Id" of the state of the unit of learning with the parts of the SCORM data model which are available.
