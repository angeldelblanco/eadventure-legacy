Assessment Profile

In <e-adventure> games it is possible to do an automatic assessment of the student interaction. The assessment profiles specifies how the assessment reports will be.

The profiles has a set of rules that evaluates certain game states (flags and vars values) and it has an specific associated actions that will be triggered when the game reach that state. Only it is possible to assign a profile for chapter, but we can support several profiles to have different possibilities of evaluation, choosing in every case which to use. So games are reusable in different contexts and for different public.
