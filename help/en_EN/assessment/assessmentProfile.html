Assessment Profile

In <e-adventure> games it is possible to do an automatic assessment of the student interaction. The assessment profiles specifies how the assessment reports will be generated.

The profiles has a set of rules that evaluate certain game states (flags and vars values) and each rule has specific actions that will be triggered when the game state it defines is reached. It is only possible to assign a profile for each chapter, but we can support several profiles to have different possibilities of evaluation, choosing in every case which to use. This way games are reusable in different contexts and for different publics.

Each evaluation profile has the following options:

  1. Profile type
  2. Student feedback configuration
  3. Assessment rules
